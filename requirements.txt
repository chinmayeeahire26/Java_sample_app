Requirement: Shopping Cart Application End-to-End Validation
1. Admin Login
The user must be able to log in using the hardcoded credentials:
Username: admin
Password: admin
On successful login, the user is redirected to the home page.
2. Product Listing
After login, the home page displays a list of products, each with a name, description, and price.
The products include at least: Apple ($1.5), Banana ($1.0), and Orange ($2.0).
3. Add to Cart
The user can add any product to the cart from the home page.
validate the correct prices are been displayed and total amount calculated correctly on cart page
4. Cart Page
The cart page lists all added products with their names and prices.
The total price is calculated as the sum of the product prices in the cart.
5. Shipping Information
The user can proceed to checkout and is presented with a shipping information form.
The form requires address, city, and postal code.
6. Order Submission
Upon submitting shipping information, an order is created and stored in a file.
The order confirmation page displays:
The order ID
The list of products and their prices 
The shipping information
7. Cart Item Management
Users should be able to remove individual items from their cart.
Users should be able to modify the quantity of each product in the cart (increase/decrease).
Users should be able to clear the entire cart at once.
The cart total price should recalculate automatically when quantities are changed or items are removed.
Cart modifications should be reflected in real-time before checkout.

Validation points:
Quantity cannot be less than 1
Removing an item completely deletes it from the cart
Total price updates correctly with quantity changes

8. Order History & Tracking
After login, users should be able to view a list of all their past orders.
Each order in the history should display:
Order ID
Order date/time
List of products and quantities ordered
Total order amount
Shipping address used
Order status (e.g., Pending, Processing, Shipped, Delivered)
Users should be able to click on an order to view full order details.
Order data should be persisted in files and retrieved on subsequent logins.

Validation points:
Orders are correctly linked to the user who placed them
Order details match what was displayed in the confirmation page
Order history is accessible from a dedicated menu/page
